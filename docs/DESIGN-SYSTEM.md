# 掉秤搭子 cut-buddy · 看板设计方案(定稿)

> 设计方向:**清新草绿**——近白底大留白、灰调墨绿文字、草绿只做点缀。
> 定稿过程:ui-ux-pro-max 引擎原始推荐"活力积木风"(因与"不焦虑陪跑"调性冲突被人工否决)→ 暖陪跑风(暖土色,用户反馈"颜色重")→ 清新草绿初版(用户反馈"太绿、不够清新")→ **当前终版**(背景去绿、文字去绿、绿色只留正反馈语义处)。
> 字体沿用 Varela Round / Nunito Sans(圆润友好,与绿色方向无冲突)。

---

## 1. 设计原则(从 PRD 调性翻译而来)

1. **清新 = 留白,不是满屏绿**:近白背景打底,绿色只出现在"该绿的地方"——趋势线、进度条、训练日、夸夸卡边框(都是正反馈语义)
2. **看趋势的视觉层级**:7 日均线永远比单日散点更醒目——视觉上就在告诉用户"看这条,别看那个点"
3. **正反馈醒目**:夸夸卡片、里程碑是看板的情绪核心,给最大的留白和唯一的彩色边框
4. **庆祝可关**:烟花等大动效必须响应 `prefers-reduced-motion`

## 2. 色彩令牌(清新草绿终版)

| 令牌 | 色值 | 用途 |
|---|---|---|
| `--accent` | `#16A34A` | 主色:图表主线、夸夸卡边框、图标描边 |
| `--accent-deep` | `#15803D` | 主色加深:页面标题、hover、进度条末端 |
| `--accent-light` | `#22C55E` | 亮绿:进度条渐变起点 |
| `--weak` | `#D7EDDE` | 弱化元素:单日散点、进度条底、分隔线 |
| `--bg` | `#FAFCFA` | 页面背景(近白,只带一丝绿意) |
| `--card-bg` | `#FFFFFF` | 卡片背景 |
| `--positive` | `#15803D` | 正反馈:趋势标签、训练日块、配额达标条 |
| `--line2` | `#0D9488` | 图表第二系列专用(如瘦体重线),与主绿区分 |
| `--text-main` | `#28382E` | 正文(灰调墨绿,白底对比度 ≈ 9:1) |
| `--text-muted` | `#5C7265` | 次要文字(对比度 ≈ 4.9:1,达标) |
| `--praise-bg` | `#F1FAF4` | 夸夸卡片专属淡绿底 |
| `--grid` | `#EAF2EC` | 图表网格线 |
| `--row-line` | `#F0F5F1` | 表格行分隔线 |

**禁用**:纯黑 `#000`、警示红(体重涨了用中性灰绿表述,不用红色制造焦虑)、AI 紫粉渐变、大面积绿底(清新感的天敌)。

## 3. 字体

| 用途 | 字体 | 回退链 |
|---|---|---|
| 标题/数字 | Varela Round | `'Varela Round', 'PingFang SC', 'Hiragino Sans GB', sans-serif` |
| 正文 | Nunito Sans | `'Nunito Sans', 'PingFang SC', 'Hiragino Sans GB', sans-serif` |

- ⚠️ **离线约束**(PRD 第 8 节):正式看板要么把 woff2 字体文件随看板本地存放,要么直接用回退链(系统中文字体本身够圆润)。**不要依赖 Google Fonts CDN**。Varela Round/Nunito Sans 只覆盖拉丁字符与数字——中文自动走 PingFang,数字(体重值)走圆体,效果协调。
- 字号:正文 ≥16px;大数字(今日体重)32-40px;行高 1.6

## 4. 布局与间距

- 卡片圆角 `16px`,内边距 `20-24px`,卡片间距 `16px`
- 柔和阴影:`0 2px 12px rgba(70, 60, 45, 0.08)`(近中性,不抢留白)
- 响应式:375px 单列 / 768px 两列 / 1024px+ 三列网格,趋势图始终通栏
- 最大宽度 `max-width: 1080px` 居中

## 5. 图表规范(Chart.js)

| 项 | 规范 |
|---|---|
| 体重趋势 | 单日散点:`--weak` 淡绿小点(弱化);7 日均线:`--accent` 草绿实线 2.5px(主角) |
| 区域填充 | 均线下方 `rgba(22,163,74,0.07)`(轻,别糊) |
| 配额对比 | 横向条:底带 `--weak`,达标填充 `--positive` |
| 身体成分 | 脂肪量 `--accent-deep` 实线、瘦体重 `--line2` 青绿虚线——**双线必须异色+异线型**,不能两条都绿 |
| 无障碍 | 线型区分(实线/虚线)+ 图例,不能只靠颜色;canvas 配 aria-label |

## 6. 动效

- 微交互 200ms,`ease-out` 进 `ease-in` 出;hover 用颜色/阴影变化,**不用 scale**(防布局抖动)
- 每屏最多 1-2 个动效元素;数字变化可做 600ms 计数动画
- 烟花彩蛋:`@media (prefers-reduced-motion: reduce)` 下降级为静态祝贺卡片
- 夸夸卡片可做一次性轻微进场(淡入上移 8px),不循环

## 7. 反面清单(这个产品尤其要避开)

- ❌ 红色/警告色表达体重上涨(制造焦虑,违背产品理念)
- ❌ 大面积绿底、绿字(就是被否决的"太绿"初版)
- ❌ emoji 当图标(用内联 SVG;文案里的 emoji 可以保留)
- ❌ 死板的纯数据表格风(PRD 反面模式:Static design / No gamification)
- ❌ 可点元素没有 cursor-pointer / 无 hover 反馈

## 8. 交付前检查清单

- [ ] 文字对比度 ≥4.5:1(本方案 text-main ≈9:1 / muted ≈4.9:1,达标)
- [ ] prefers-reduced-motion 已处理(尤其烟花)
- [ ] 375/768/1024/1440 四档无横向滚动
- [ ] 图标统一内联 SVG(Lucide 风格,24×24 viewBox)
- [ ] 字体离线可用(本地 woff2 或回退链)
- [ ] 图表色盲友好(异色+异线型+图例)
