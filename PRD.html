一句话：基于 B 站「好人松松」减脂方法论的个人减脂/增肌陪跑搭子——你用大白话报体重 / 饮食 / 运动，它替你结构化存档、按松松配额给建议、用「看趋势不看单日」判断你是不是真在变好，并生成一个本地可视化看板陪你坚持下去。

1产品定位

是什么：一个装在本地、用对话驱动的减脂 / 增肌过程管理工具。

理念三条

方法论 = 好人松松：碳蛋脂配额、缺口、碳水递减、两周趋势反馈、不掉秤排查。
看趋势，不看单日：用 7 日移动平均 + 自然周周均判断"真减脂 / 真增肌"，不被单天的水分波动吓到。
不制造焦虑的陪跑语气：减脂是方法问题，不是意志问题；只给可执行的建议，不羞辱、不极端。

全程本地：数据存在你自己电脑，看板是本地网页，不联网、不上传、不做公开页面。

2目标用户

  • 想科学减脂或增肌、愿意每天用一句话打卡的人。
  • 认同"看长期趋势、不跟单日体重较劲"的人。
  • 不适用：需要医疗诊断 / 伤病康复处方 / 孕期等特殊情况——这些请找医生，本工具不替代专业医疗。

3能力清单

先看三大板块，再细说每块：

你说人话，我帮你记账算营养
🧭
科学指导
按「好人松松」给你配额、判趋势、出建议
🤝
看板陪伴

（建档、导入历史等"上手步骤"见第 4 节流程。）

📝 随手记录 · 说人话就帮你算账

随口说"中午吃了碗牛肉面""早上称了 120 斤"，它自动结构化存档、算出热量和碳蛋脂；混合菜拆开算更准；你常吃的食物会被记住（给过一次营养后，下次报名字就行）。只记已发生、不记计划（"打算去练腿"不入库，练完再记）。

🧭 科学指导 · 按「好人松松」配额与建议

  • 按 性别 × 阶段（减脂 / 增肌） 算碳蛋脂配额区间（下限 – 目标 – 上限），告诉你这顿该补什么、少吃什么；
  • 用 7 日均 + 周均判断你是不是"真在变好"（不被单日体重骗）；
  • 平台期、降太猛会帮你排查提醒。全程不焦虑陪跑。

🤝 看板陪伴 · 可视化看板 + 主动提醒

看板不只是"数据展示"，更是掉秤搭子陪伴你的方式——把抽象的努力变成看得见的进展，用持续的正反馈托住你、让你更想坚持。

看板模块（单个本地 HTML，每次记录自动刷新）：

  • 里程碑进度条：从起点到各阶段目标，你走到哪一步；达成目标那天刷新看板会有全屏小烟花 🎉，恭喜你、并邀你设下一个里程碑。
  • 今日状态：今日体重、7 日均重、趋势标签 + 一句陪跑。
  • 体重趋势图：每日散点 + 7 日均线。
  • 饮食日记 + 配额区间：每餐吃了啥、热量碳蛋脂，对照配额区间。
  • 活动日历：这个月哪几天练了什么。
  • 身体成分：近 4 周脂肪量、瘦体重趋势。
  • 今日份夸夸 ⭐：每天一句基于你真实数据的真诚夸奖（看板上做成醒目的暖色卡片）——给情绪的正反馈，减脂路上你需要有人看见你做得好。
  • 本周小结：本周周均、训练天数、一句趋势判断。

主动提醒：聊天时主动问你今天记全没；也可选每晚定时提醒打卡（需 Claude Code 定时任务）。

后续版本 · 饮食复盘 —— 数据攒够后出周 / 月碳蛋脂达标率、常吃食物排行、饮食 ↔ 体重趋势关联。

4首次使用旅程

1
建档填资料（含怎么称呼你 → 用作看板标题、目标体重 → 自动建里程碑）+ 历史体重三选一（导入苹果健康 / 传截图 / 跳过）
2
记第一条数据手把手引导你记第一笔
3
立刻生成看板第一条记完马上出图，给即时正反馈 🎉
4
之后每次记录，看板自动刷新同一个看板文件，始终是最新的你

💡 可选小建议：把看板设成浏览器首页 / 固定标签（本地 file:// 路径），打开就能随手瞄一眼——推荐，但不强制。

5数据与隐私

  • 所有个人数据存在用户本地数据目录，不联网、不上传。
  • 每个用户档案彼此独立；分发的 skill 包不含任何他人个人数据。
  • 个人常吃食物库、体重 / 饮食 / 运动记录均为本地文件，用户可自行查看与备份。

6不做什么（Non-goals）

  • 不做医疗诊断、不开伤病康复处方、不推荐减肥药 / 代餐智商税。
  • 不做公开可访问的网页、不做社交分享后台。
  • 不做"局部减脂""三天瘦十斤"等违背科学的承诺。

7健康与安全护栏

  • 缺口温和（10–20%），不鼓励极端节食；碳水有 2g/kg 地板，蛋白保肌优先。
  • 单日波动不下结论，平台期先排查执行再谈调整。
  • 涉及经期 / 激素水肿等松松未覆盖的内容，用通用循证解释并注明"非松松内容"，不伪托。
  • 任何疑似医疗问题，明确建议就医。

8技术形态（概要）

  • 引擎：Python 标准库（无第三方依赖），数据存本地 CSV / JSON。
  • 看板：单 HTML + 本地 Chart.js（离线可用）。
  • 苹果健康导入：解析导出的健康数据，清洗异常值。
  • 运行环境：为 Claude Code 设计——即装即用、自动触发、可选定时提醒，体验最完整。引擎本身是通用 Python，在 Codex 等其它 agent 上也能跑脚本、出看板，但「自动触发」「定时提醒」可能用不了、需手动配合。建议优先在 Claude Code 上使用。
